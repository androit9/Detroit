
Описание проекта
Данное приложение представляет собой клиент для Windows, написанный на C/C++. Приложение работает в фоновом режиме и использует протокол TCP для связи с сервером.

Выполненные операции
Тихий запуск:

Реализован вызов FreeConsole(), что позволяет скрыть консольное окно при запуске приложения.
Реализовано но нужно вмешательство конечного пользователя: Автоматический запуск приложения при входе пользователя в систему. Для этого необходимо добавить запись в реестр Windows или создать ярлык в папке автозагрузки.
Работа в фоновом режиме:

Использование стандартной библиотеки std::thread для создания фонового потока, который выполняет задачи захвата скриншотов и их отправки на сервер. Это позволяет приложению работать в фоновом режиме.
Общение с сервером:

Реализовано общение с сервером с использованием сокетов по протоколу TCP. Приложение отправляет захваченные скриншоты на сервер и получает данные.
Отсутствие сторонних библиотек:

Приложение использует только стандартные библиотеки C++ и Windows API. Не было использовано никаких сторонних библиотек или фреймворков, таких как Boost или Qt.
Клиентская часть:

Код написан на C++, что соответствует требованиям технического задания.
Проверка активности пользователя:

Реализована функция для проверки активности пользователя. Приложение отправляет информацию о системе, включая статус активности пользователя, IP-адрес и имя хоста.
Ограничения и недоработки
Автоматический запуск: Необходимо реализовать функциональность для автоматического запуска приложения при входе пользователя в систему. Это можно сделать через:

Запись в реестр Windows в ключ HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run.
Создание ярлыка в папке автозагрузки (C:\Users\<Имя_пользователя>\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup).
Обработка ошибок: В текущей версии приложения не реализована полноценная обработка ошибок. Рекомендуется добавить дополнительные проверки и логирование для улучшения стабильности приложения.

Интерфейс пользователя: Приложение не имеет графического интерфейса, так как оно должно работать в фоновом режиме. Однако, для удобства пользователя можно рассмотреть возможность реализации простого интерфейса для настройки параметров.

Заключение
Данное приложение соответствует большинству требований технического задания, однако требует доработки для автоматического запуска при входе пользователя в систему. Рекомендуется провести тестирование и улучшение функциональности для повышения надежности и удобства использования.
